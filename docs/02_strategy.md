# session-scribe 戦略書(周知・マネタイズ)

## 前提

このプロダクトはClaude Skillとして無料配布する。初期マネタイズは狙わず、**ブランド構築とユーザー基盤獲得を優先**し、Phase 2以降で付加価値SaaSとして回収する。

## 周知戦略

### 前提となる難しさ

対象ユーザーは「AI活用している × 発信したい × でも面倒」という3条件が揃った層で、母数はあるが顕在化していない。広告は効きにくく、コンテンツマーケティングと口コミ導線が本命。

### フェーズ1: 初期100ユーザー獲得(0〜3ヶ月)

#### ① ドッグフーディング + 成果物そのものが宣伝になる構造

- ツールで生成した記事の末尾に「この記事はsession-scribeで生成されました」のフッター(ユーザーON/OFF可)を入れる
- 生成された記事がZenn/Qiitaに流れるたびにそれが広告になる
- Yuto自身がホームラボ記事を週1〜2本、このツールで生成して公開
- ポートフォリオサイトでも「このツールで書いた」と明示
- 3ヶ月で「このツール使って書いた記事」が検索結果に並ぶ状態を作る

#### ② ターゲットコミュニティへの直接ノック

- Zennのtrending上位に出るAI系記事の著者に個別連絡
- X(Twitter)のAIコーディング系発信者(claude-code、Cursor、Cline等のタグ)にリプライ
- Discordコミュニティ(ホームラボ系、LLM活用系)での告知
- 初期は個別ノック50件の方が、バズ狙い1回よりROIが高い

#### ③ 「作ってみた」記事を自分で書く

- Zennで「Claude Codeのログを記事化するSkillを作った話」を公開
- 技術記事としてもツール紹介としても機能する一石二鳥
- Yutoのポートフォリオ強化にも直結

#### ④ Anthropic公式エコシステム狙い

- anthropics/skillsリポジトリへのPR検討
- Plugin Marketplace / Anthropic Official Marketplaceへの掲載
- 公式マーケットプレイス掲載は大きなシグナリングになる

### フェーズ2: 1,000ユーザー(3〜9ヶ月)

#### ① SEO勝負に切り替える

- ユーザーが生成した記事が「Claude Code エラー 〇〇」等のロングテール検索で上位を取り始める
- ツールの存在より、ツールで生成された記事群が検索流入の入り口になるフェーズ

#### ② インフルエンサー配布

- AIコーディング系の影響力ある発信者に無料プロ版提供
- 依頼せず「使ってみてください」で渡し、感想を自然投稿してもらう
- 依頼投稿は逆効果なので避ける

#### ③ 技術カンファレンス・LT

- builderscon、技術書典、LT会での発表
- 「AI時代のナレッジマネジメント」文脈で登壇すると企業担当者に刺さる

### フェーズ3: スケール(9ヶ月〜)

- 企業採用が見え始めるフェーズ
- 情シス・DX推進担当者向けのウェビナー
- ITmedia等メディア露出
- Product Hunt投稿(英語版)

### 避けるべき罠

- Qiita/ZennにSEO狙いで量産記事を書く → 品質低下で逆効果
- 有料広告(X広告/Google広告)の早期投入 → LTVが見えない段階では燃やすだけ
- StartupDB系 → 日本のtech系新規ユーザー層には刺さらない

## マネタイズ戦略

### 基本構造

Skill無料配布でブランド・ユーザー基盤を作り、Phase 2で付加価値SaaSとして回収する。

### Phase 1(Skill、0〜6ヶ月): 認知獲得・無料配布

- GitHub公開、Anthropic Marketplace掲載狙い
- 目的はマネタイズではなくブランド構築
- 「あの便利なSkillを作った人」というポジション確立
- Yuto自身のキャリア(AI/LLM転職)への強いシグナリング

### Phase 2(Webアプリ/SaaS、6〜12ヶ月): マネタイズ開始

Skillで溜まったユーザー基盤に対して、以下を有料提供:

- **クラウド同期**(複数マシンで蓄積を共有)
- **Team機能**(社内ナレッジベース)
- **Notion/Confluence自動投稿**
- **横断検索Webアプリ**(ベクトル検索対応)
- **バックアップ・エクスポート機能**

Skillは無料のまま、付加価値Webサービスで課金する二段構え。

#### プラン設計案

| プラン | 価格 | 内容 |
|---|---|---|
| Free | $0 | Skill単体。ローカル完結。 |
| Pro | $3〜5/月 | クラウド同期、マルチデバイス、バックアップ |
| Team | $20/席/月 | Notion/Confluence連携、チーム内検索、管理ダッシュボード |
| Enterprise | 要相談($500〜/月〜) | オンプレ、SSO、監査ログ、カスタムマスキング |

### Phase 3(Enterprise、12ヶ月〜): 本格収益化

- 企業向けオンプレ、SSO、監査対応
- 月額数万円〜のエンタープライズプラン
- **1社月5万円 × 20社 = 月100万円**が最初の現実的マイルストーン
- 個人Pro課金で同額を目指すと1,000〜2,000人必要で遥かに難しい

### 本命はBtoB

企業で刺さる理由:

- **AI活用の属人化問題**: エンジニアがAIで解決した内容が本人の頭にしか残らない問題は、マネジメント層の課題感が高い
- **セキュリティ要件**: マスキング+オンプレ対応は中堅以上の企業で必須
- **予算枠**: ナレッジマネジメント予算、DX推進予算、情シス予算のいずれにも乗せやすい

### マネタイズで避けるべき罠

- 最初から有料化 → ユーザーが集まる前の課金は母数を減らすだけ
- 安売り競争(月額$1〜3プラン) → 人が流れてTeam/Enterpriseの導線が死ぬ
- 個人サブスクだけで食おうとする → 日本市場の個人課金文化は弱い

### 副次的な収益源(参考、本命ではない)

- Zenn Books/技術書典: 「AI時代の個人ナレッジマネジメント」本
- GitHub Sponsors、Zennスポンサー
- Anthropic API等へのアフィリエイト

## KPI設計(参考)

| 時点 | 登録ユーザー | 月間生成記事 | 公開記事 | 有料転換 | 法人契約 |
|---|---|---|---|---|---|
| 3ヶ月 | 100 | 300 | 100 | - | - |
| 6ヶ月 | 500 | 1,500 | 500 | - | 問い合わせ1〜2件 |
| 12ヶ月 | 2,000 | 6,000 | 2,000 | 2%(40人) | 3〜5社 |

**最重要KPIは「公開記事数」**。公開されるほど宣伝が広がる構造のため。

## 専門家としての本音

成否を分ける3点:

1. **生成品質**(SKILL.mdのプロンプト設計)
2. **最初の6ヶ月の地道なコミュニティ活動**
3. **企業版への舵切りタイミング**

特に②は多くの個人開発者が軽視する。Yuto自身がホームラボで情報発信を継続することが、最強の周知戦略。

ポートフォリオサイトも既にあるので、**「このツールで書いたホームラボ記事群」が検索に載る状態**を半年で作れれば、そこが資産になる。
