# session-scribe プロダクト定義書

## 一行説明

AIとの対話を、自動で検索可能な個人ナレッジに変える。必要なら記事として公開も。

## 背景・問題意識

Claude Code / Codex / Copilot等のAIコーディングツールを使ったトラブルシュートの知見は、本人の頭と対話ログに残るだけで活用されていない。記事化したい気持ちはあっても、後から書き起こす心理的・時間的コストが高く、9割は埋もれている。

本プロダクトは「AIセッションが終わったら、勝手にドラフトが自分の保管庫に溜まっていく。公開するかどうかは後で選ぶ」というUXで、この課題を解消する。

## コアコンセプト

**「AIセッション → 自動で構造化されたナレッジ」** という変換レールを最初に敷くプロダクト。

- 発信したい人だけでなく、「AIを使い倒している人」全員が第1ターゲット
- 発信しない人にとっては「第二の脳(Second Brain)」として機能
- 発信したい人にとっては「公開用ドラフトが勝手に溜まる場所」

## ターゲットユーザー

### 第1ターゲット(MVPで狙う)

- Claude Code / Claude Desktop を日常使用する個人エンジニア
- 自分の作業履歴を後から参照したい人
- ホームラボ、個人開発、学習中など、技術的な試行錯誤が多い人

### 第2ターゲット(ローンチ後に拡張)

- Zenn/Qiita等で発信している技術者
- チーム内ナレッジ共有したい小規模開発チーム

### 第3ターゲット(マネタイズの本命、6ヶ月後以降)

- 企業のDX推進担当者、情シス、エンジニアリングマネージャー
- 「AI活用で属人化した知見を組織資産にしたい」ニーズ

## UX(想定フロー)

```
ユーザー(Claude Code内で):
「今日のセッション、ナレッジ化しておいて」
       ↓
Claude Code:
[SKILL.mdを自動ロード]
[スクリプトを実行してセッションログをパース]
[ログ内容を自分で読んで構造化]
[マスキングスクリプトを実行]
[~/knowledge-log/ に Markdown保存]
「3件のセッションを記事化しました。保管庫に保存してあります」
```

「公開」判断は後から手動で行う。心理的ハードルが分散され、意思発動なしに資産が溜まる構造。

## コア機能(MVP)

1. Claude Codeセッションログの読み込み(`~/.claude/projects/`配下のJSONL)
2. LLMによる構造化(課題/試行/解決)
3. マスキング(APIキー、内部IP、メールアドレス等の自動検出)
4. Markdownとしてローカル保管庫に保存
5. 手動トリガー(Claude Codeで「記事化して」と依頼)

### 出力フォーマット(目標)

```markdown
# [タイトル: LLMが生成]

## 環境
- OS / 言語 / ライブラリバージョン(セッションから抽出)

## 発生した問題
[何が起きたか、エラーメッセージ等]

## 試したこと
1. [アプローチ1] → 失敗理由
2. [アプローチ2] → 失敗理由
3. [アプローチ3] → 成功

## 解決方法
[最終的な解決策とコード]

## 学び
[なぜこの解決策が効いたか、次回への教訓]
```

## MVPで「やらない」こと(重要)

- 完全自動のセッション完了検知 → 最初は手動トリガー
- Codex/Copilot対応 → Claude Codeのみ
- Zenn/Qiita API直接投稿 → まずはMarkdownコピーで十分
- チーム機能、Notion連携 → 後回し
- ベクトル検索 → 全文検索も不要、まずはファイル一覧で十分
- クラウド同期 → ローカル完結
- Webダッシュボード → 不要(Markdownをエディタで見ればよい)
- ユーザー認証・課金機能 → ローカルツールなので不要

## 差別化の核

現時点で大手が提供していない価値:

- Claudeの「Projects」: プロジェクト内の文脈共有はできるが、過去セッションの検索・ナレッジ化は弱い
- GitHubの「Copilot Workspace」: コーディング特化で、ナレッジ蓄積の視点がない
- Obsidian等のPKM: 手動入力が前提

→ **「AIセッション → 自動ナレッジ化」というレールを最初に敷いたプロダクト**になれる可能性がある。

## プロダクト形態の決定

**Claude Skill 単体**としてMVPを作る。

### なぜSkillか

- LLM API料金が不要(ユーザーのClaude契約で動作)
- 開発コストが劇的に低い(SKILL.md + 補助スクリプト)
- Claudeのエコシステムに乗れる(先行者利益)
- 配布が楽(GitHub配布、将来的にMarketplace)
- オープン標準化されており、将来Claudeに縛られない

### 将来の拡張

- Phase 2(3〜6ヶ月後): MCPサーバーを追加し、検索・投稿機能を強化
- Phase 3(6ヶ月後〜): 独立Webアプリ + BYOK方式でClaude以外のユーザーへ拡張

## 成功指標(ローンチ後3ヶ月)

- GitHub Star: 200+
- 実ユーザー(ダウンロード後1週間継続使用): 50+
- 自身のドッグフーディング: 週2セッション以上を3ヶ月継続
- Anthropic公式Marketplace掲載(努力目標)
